# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有草莓芽芽。
- 草莓芽芽承担核心动作，不只是装饰。
- 草莓芽芽第一眼像桌宠 spritesheet 默认形象，而不是泛草莓角色。
- 草莓芽芽必须保留核心识别组合：圆鼓鼓草莓壳帽、额前叶片刘海、小桌宠比例、浅绿色连体装、红色小鞋。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面轻盈、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 草莓芽芽像头像、贴纸、表情包或单纯可爱卡通。
- 草莓芽芽像草莓帽小女孩、普通萌妹、长身人物或儿童绘本角色。
- 草莓芽芽缺少圆鼓鼓草莓壳帽、额前叶片刘海、绿色连体装、红色小鞋等识别点。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有轻巧的视觉隐喻。
- 和 `assets/examples/` 里的旧案例构图过于相似。

## 迭代方法

- 太普通：让草莓芽芽成为动作主体，加入一个轻巧但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 structure-first、not sticker poster、not mascot-only，减少装饰。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和角色动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先认出“草莓芽芽在认真做一件小事”，然后 1 秒内看懂结构。

如果第一眼像教程页、贴纸页或儿童海报，而不是白纸上的轻量产品草图，就不合格。
