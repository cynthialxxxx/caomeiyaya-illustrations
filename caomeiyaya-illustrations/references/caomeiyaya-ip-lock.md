# 草莓芽芽 IP 锁定规则

## 主锚点

草莓芽芽的最终形象以桌宠素材 `assets/caomeiyaya-reference/spritesheet.webp` 为唯一主锚点。

`assets/caomeiyaya-ip-lock/caomeiyaya-ip-model-sheet.png` 是当前定稿板，生成正文配图、样片或新动作时，应先满足这张图里的比例、轮廓和识别规则，再考虑正文隐喻。

## 第一眼识别

合格的草莓芽芽必须第一眼像桌宠素材里的小玩偶，而不是泛草莓角色、草莓帽小女孩或儿童绘本人物。

必须同时保留：

- 圆鼓鼓的草莓壳帽，帽体饱满，有浅色籽点。
- 额前贴着脸的绿色叶片刘海，形成明显叶冠。
- 极高头身比，短手短脚，小桌宠比例。
- 浅绿色连体装或背带装，红色小鞋。
- 圆脸、亮眼、粉腮，表情软萌但有动作意图。
- 厚黑描边和 Q 版玩偶感。

## 允许变化

为了服务正文配图，草莓芽芽可以：

- 坐、跑、推、拉、搬、修补、踩机器、拉藤蔓、抱纸片。
- 表现努力、困惑、委屈、开心、惊讶、摔倒等状态。
- 使用草莓藤蔓、小篮子、果酱罐、叶片标签、小工具箱等道具。
- 适度降低色彩饱和度，让角色融入白底手绘解释图。
- 让道具承担信息结构，例如输入、输出、反馈、断点、桥梁、筛选。

## 禁止跑偏

不要出现：

- 草莓帽小女孩、长身卡通人物、普通萌妹。
- 普通红色兜帽、头盔、蘑菇帽或草莓发饰替代草莓壳帽。
- 丢失叶片刘海、红色小鞋、浅绿色连体装等关键识别点。
- 轻水彩儿童绘本、商业海报、精致扁平插画、3D 渲染、写实玩偶。
- 角色站在角落旁观，正文结构与角色动作无关。
- 画面去掉草莓芽芽后仍然完全成立。

## 生成提示词固定块

每次生成或编辑草莓芽芽时，提示词必须包含等价约束：

```text
角色以桌宠素材里的草莓芽芽为唯一形象锚点：圆鼓鼓草莓壳帽、额前绿色叶片刘海形成叶冠、极高头身比、短手短脚、小桌宠比例、浅绿色连体装、红色小鞋、圆脸亮眼粉腮、厚黑描边、Q版玩偶感。不要画成草莓帽小女孩、长身人物、普通萌妹、儿童绘本水彩角色或泛草莓角色。草莓芽芽必须承担画面核心动作。
```

## 验收问题

生成后逐项检查：

1. 第一眼是否像桌宠 spritesheet，而不是重新设计的草莓角色？
2. 草莓帽是否是圆鼓鼓外壳，而不是普通兜帽？
3. 叶片刘海是否清楚贴在额头上？
4. 头身比是否仍是小桌宠比例？
5. 去掉草莓芽芽后，图的核心动作是否会失效？
6. 文字、箭头和结构是否服务正文，而不是把图变成海报？
