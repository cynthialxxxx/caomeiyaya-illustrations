# 草莓芽芽 IP

## 角色定义

草莓芽芽是这套中文正文配图 Skill 的固定视觉 IP。

默认每张图都要出现草莓芽芽。草莓芽芽不是贴纸、头像或站在旁边的可爱装饰，而是正在认真参与系统运转、搬运信息、解决卡点、让抽象概念变得可见的小桌宠。

## 外形

- 圆润幼态比例，大头小身，动作夸张但线条干净。
- 红色草莓帽是最高识别点：帽子带浅色籽点，边缘像草莓果肉。
- 额前有明显绿色叶片刘海，头顶可有小叶芽。
- 两侧可出现草莓藤蔓、小草莓挂坠或草莓耳坠，但不要堆满。
- 穿浅绿色背带装或连体装，可有红色纽扣、小花口袋、红色鞋。
- 脸部柔软，有腮红；表情可以困惑、委屈、开心、努力、惊讶、摔倒，但要服务画面动作。
- 手绘黑色线稿为主，红色与绿色只用于角色识别，避免变成厚涂卡通海报。

## 性格

- 软萌、鲜活、陪伴感强，但不只是卖萌。
- 做事认真，常常用小小身体承担很大的信息任务。
- 有一点笨拙和戏剧化：会被流程绊倒、抱着材料奔跑、趴在机器边缘检查问题。
- 情绪表达比小黑更明显，但每个表情都要帮助读者理解文章里的状态。
- 像一个把复杂工作流变轻、把卡点变可见的草莓小助手。

## 常见职责

让草莓芽芽承担核心动作：

- 抱着素材草莓、纸团、便签或小箱子奔跑。
- 用草莓藤蔓拉住信息线、路径线或反馈线。
- 趴在断点边缘，把断掉的流程重新接起来。
- 在低科技机器里踩踏板、推拉杆、拧阀门、观察表盘。
- 把混乱内容分拣到小篮子、果酱罐、抽屉或邮筒里。
- 把一个大想法压成几颗小草莓，表示复用、拆分或沉淀。
- 举小牌提醒风险，但牌子文字要短。
- 在地图路线里跌跌撞撞前进，最终把内容递到出口。
- 用叶片、藤蔓或小草莓做轻量视觉连接，表达输入、反馈、回流。

## 禁止

- 不要只画成可爱头像、表情包、贴纸或壁纸。
- 不要画成商业 IP 海报、儿童绘本封面、厚涂插画或精致扁平插画。
- 不要堆满草莓、花朵、爱心、闪光和装饰背景。
- 不要让草莓芽芽只是站在角落里看。
- 不要让角色抢走结构表达；可爱必须服务认知动作。
- 不要把角色画丢关键识别点：草莓帽、叶片刘海、绿色背带装至少保留两项。

## 判断标准

如果去掉草莓芽芽，图的核心隐喻还能完全成立，说明草莓芽芽太装饰了；要重写提示词，让她成为动作主体。

如果第一眼只觉得“可爱”，但看不出她正在解决什么问题，也不合格。合格的草莓芽芽配图应该先有角色识别，再有结构表达，最后留下一个轻巧的小记忆点。
