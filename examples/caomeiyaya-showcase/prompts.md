# 草莓芽芽正文配图样片 Prompts

这些 prompt 对应 `examples/caomeiyaya-showcase/` 里的 6 张样片。当前正式样片是 IP 锁定版：角色直接来自桌宠 spritesheet 默认帧，结构部分由 `scripts/generate_ip_locked_showcase.py` 生成；下面的 prompt 用于后续继续用图像模型重绘或微调。

## 01 信息过载

```text
Use $caomeiyaya-illustrations 为“信息过载时，先接住，再分拣可用线索”生成一张 16:9 中文正文配图。
结构类型：系统局部。
画面：左侧有评论、链接、截图、灵感、想法等小纸片飞来，中间是一个低科技漏斗，草莓芽芽守在漏斗下方努力接住信息，右侧输出一个标着“可用线索”的小盒子。
中文标注：信息过载 / 先接住 / 再分拣 / 可用线索。
要求：白底、黑色手绘线稿、大量留白、少量橙色箭头、草莓芽芽必须是核心动作主体。
```

## 02 产品验证

```text
Use $caomeiyaya-illustrations 为“产品验证不是一次做大，而是先把假设推过一扇小门”生成一张 16:9 中文正文配图。
结构类型：Workflow。
画面：左侧是“假设”盒子，中间是一扇窄窄的“验证门”，草莓芽芽正用力把假设推过门，右侧是“证据”盒子。
中文标注：假设 / 验证门 / 证据 / 能过再放大。
要求：不要正式流程图，不要课程页；画面清爽，草莓芽芽要承担推动验证的动作。
```

## 03 内容复利

```text
Use $caomeiyaya-illustrations 为“一颗想法可以长成多次使用的内容资产”生成一张 16:9 中文正文配图。
结构类型：概念隐喻。
画面：草莓芽芽在左下角给一株内容小植物浇水，植物枝头长出“文章”“短视频”“案例”“脚本”等小果实，右侧有一个“素材库”盒子接收成熟产物。
中文标注：内容复利 / 一颗想法 / 长成多次使用 / 素材库。
要求：白底、留白、轻巧、有戏，不要画成营销海报。
```

## 04 信任建立

```text
Use $caomeiyaya-illustrations 为“信任不是喊出来的，而是一块证据一块证据铺过去”生成一张 16:9 中文正文配图。
结构类型：地图路线。
画面：画面中间是一条小裂谷，草莓芽芽把“案例”“截图”“数据”“反馈”“承诺”几块石头铺成桥，读者可以从左走到右。
中文标注：信任建立 / 搬证据 / 放心通过。
要求：草莓芽芽必须搬证据石头，不要只是站在桥边。
```

## 06 自动日报

